SOFTWARE REQUIREMENTS SPECIFICATION (SRS)
1. INTRODUCTION
1.1 Purpose

This document defines the complete functional and non-functional requirements for the Hair Salon Appointment & Live Queue Management System, a cross-platform mobile application built using Expo (React Native) with backend services hosted on Railway.

The system enables:

Clients to book appointments and view real-time queue status.

Admin to manage appointments, schedule availability, and live queue operations.

1.2 Scope

The application will:

Support Android & iOS using Expo.

Provide role-based access (Admin / Client).

Allow appointment booking with dynamic time slot management.

Maintain a real-time queue system.

Provide daily session management (open/close).

Allow drag-and-drop queue reordering.

Provide telephonic quick-call integration.

Be scalable to support multiple salons in future (Phase 2).

1.3 Definitions
Term	Description
Slot	Fixed duration time interval for booking
Session	A single operational day of the salon
Queue	Ordered list of active appointments
Active Day	A date marked open for booking
Live Queue	Real-time ordered appointment list

3. FUNCTIONAL REQUIREMENTS
3.1 Authentication & Authorization
FR-1: User Registration

Username must be 10-digit phone number (unique).

Password (minimum 8 characters).

First name

Last name

Profile image (optional)

Role (assigned by admin only)

FR-2: Login

JWT-based authentication

Role-based route protection

Token expiration + refresh mechanism

FR-3: Profile Management

Both roles can:

Change first name

Change last name

Change password

Change profile image

Phone number (username) cannot be changed.

3.2 Admin Functional Requirements
FR-4: Manage Users

Create users (Admin or Client)

Delete users

View list of users

Deactivate user

Constraints:

Cannot delete self

Cannot delete super-admin (if exists)

FR-5: Manage Salon Schedule

Admin can:

Mark specific calendar dates as:

Open

Closed

Holiday

Define default working hours (e.g., 9AM–6PM)

Override working hours for specific dates

Define slot duration (e.g., 15 / 20 / 30 minutes)

Changing slot duration:

Only allowed if no future bookings exist

Otherwise require confirmation

FR-6: Appointment Management

Admin can:

View all appointments (calendar + list view)

Filter by:

Date

Status (Booked / Completed / Cancelled / No-show)

Edit appointment time

Delete appointment

Reassign appointment to new slot

FR-7: Drag & Drop Queue Reordering

Admin can:

Touch-hold an appointment

Drag up/down to reorder

Save queue order

System Behavior:

Queue position auto-updates

Order index stored in database

Real-time update reflected to clients

FR-8: Live Queue Management

Admin can:

Mark appointment as “In Service”

Mark appointment as “Completed”

Mark appointment as “No-show”

Close today’s session (only if queue empty)

When session closed:

No new bookings allowed

Today's session archived

FR-9: Direct Call Feature

Admin can:

Tap appointment

Press “Call Client”

Device dialer opens with phone number

3.3 Client Functional Requirements
FR-10: View Available Days

Client can:

See open days

See working hours

See remaining slots

Closed days not selectable.

FR-11: Book Appointment

Client:

Select date

Select available slot

Confirm booking

System:

Lock slot

Prevent double booking

Assign queue position

Send confirmation notification (optional Phase 2)

FR-12: Cancel Appointment

Client can:

Cancel own future appointment

Cannot cancel past or completed appointments

Cancellation Policy:

Optional: Prevent cancellation within X hours before slot

FR-13: View Live Queue

Client can:

See:

Current serving

Their queue position

Estimated waiting time

Auto-refresh every X seconds

Real-time via WebSocket (preferred)

4. SYSTEM WORKFLOW
4.1 Booking Flow

Client → Select Date → View Slots → Confirm → Slot Reserved → Queue Assigned

4.2 Queue Flow

Booked → Waiting → In Service → Completed

or

Booked → Cancelled / No-show

4.3 Daily Session Flow

Admin Opens Day → Bookings Active → Manage Queue → Complete All → Close Session
